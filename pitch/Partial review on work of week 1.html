# Moving Average
def MA(seq, w):
    """
    Moving average
    seq: sequence; w: window size
    """
    out0 = np.convolve(seq,np.ones(w,dtype=int),'valid')/w
    r = np.arange(1,w-1,2)
    start = np.cumsum(seq[:w-1])[::2]/r
    stop = (np.cumsum(seq[:-w:-1])[::2]/r)[::-1]
    return np.concatenate((  start , out0, stop  ))

This plot kind of shows the main framework in my mind so far. (For simplicity, many other plots are omitted)

The details are as follows:

Intro¶

  • The blue line is the observed pitch, green line the robot velocity, orange line the robot acceleration.

  • I shifted the observed pitch to erase the potential bias.

  • I located the symmetric track and based most of my analyses on them.

observation¶

  • The abrupt change of velocity corresponds to large value of acceleration (for sure), which would lead to larger amplitude of observed pitch in general.

  • Large value of acceleration with no obvious change in velocity might suggest a huge bumper which causes greater jitters in our observation.

  • The track we travelled is strongly symmetric, which can be used in later analyses to identify the true pitch.

Framework¶

  • quantify the effect of acc

  • filter out the effect of motion of vehicle

  • filter out the bumper effect (ideally would be symmetric) by a low pass filter

  • Interpolate/estimate the pitch of asymmetric part.
